### B. 구독 환불 정책 - 사례 스터디 후 확정
- **이슈**: 구독은 월 결제 vs 년 결제 금액이 다름 (년 결제가 할인). 중도 환불 시 어떻게 계산?
- **할 일**: 숨고·크몽 등 사례 스터디 → 우리 정책 확정 → 약관 수정
- 검토 포인트: 년 결제 중도해지 시 ①월 정가 기준 사용분 차감 후 환불 ②일할 계산 ③위약금 여부
- 현재 약관(제8조 ⑤): "구독은 7일 이내 미사용 전액환불, 사용 후 잔여기간 일할계산" → 월/년 구분 없음, 보완 필요

### C. UX/기능 개선 아이디어
**C-1. 열람 시 금액 지출 표시** — 파트너가 열람할 때 "공개 견적 1만원 / 지정 견적 5만원이 차감되었습니다" 명확히 표시 (현재 토스트는 단순)
- 열람 전 확인(이 견적 열람 시 ○○원 차감됩니다) → 동의 후 차감 흐름도 검토

**C-2. 가격 정책 안내 페이지 (파트너 전용)** — 열람가/구독료를 파트너에게 안내하는 곳 필요
- ⚠️ **중요**: 고객에게는 노출되면 안 됨 (파트너만 보는 페이지)
- 위치 후보: 파트너 대시보드 내 "요금 안내" 또는 지갑 탭

**C-3. 견적 종료 버튼** — 고객이 마음 바뀌거나 파트너 정해지면 견적 종료
- 고객 마이페이지에서 견적 "종료/취소" 버튼
- 검토: 당근마켓식 "거래완료" 시스템 필요? (파트너 선택 → 거래 확정 → 완료 → 리뷰)
- 상태 흐름: registered → matched(파트너선택) → completed(완료) 활용
- 종료 시 진행중이던 다른 파트너 채팅·견적 처리 방안도 고려

**C-4. 고객 마이페이지 견적에 지정/공개 뱃지** — 본인이 의뢰한 견적이 지정인지 공개인지 뱃지 표시 (간단, 빠르게 가능)

- 대분류 4개(식당/가정/공장/상업) + 소분류 9개
- 관리자 순서변경, 활성토글(임시숨김)/하드삭제(영구) 구분
- 공용 컴포넌트 `CategorySelect.tsx`

### 지역 선택 (2단계)
- 시/도 → 구/시/군, 전국 행정구역 (`RegionSelect.tsx`)
- 순서: 서울 → 광역시 → 도 → 세종/제주

### 견적 의뢰
- 카테고리/지역 2단계 선택, 사진 첨부(최대 3장), 필수값 검증
- 공개 견적 / 지정 견적(특정 파트너 지목) 구분

### 파트너
- `PartnerAvatar.tsx`: 로고 또는 회사명 첫글자 컬러 아바타
- 마이페이지: 파트너=정보수정 / 일반=견적목록
- 파트너는 견적의뢰·파트너찾기 차단 (흐리게+안내)
- **사업자등록증 첨부 필수**
- **국세청 사업자번호 인증** (진위/휴폐업 확인, NTS API)

### 회원가입/약관
- 단순 가입(고객/파트너 구분 제거), 카카오/네이버/구글 버튼
- 약관 페이지: `/terms`(일반), `/privacy`(개인정보), `/marketing`(마케팅), `/partner-terms`(파트너 전용)
- 파트너 등록 시 개인정보 중복동의 제거 (가입 시 이미 동의)
- 약관 임시값(사업자번호 등) 빨간색 표시 → 오픈 전 수정용

### 지갑 시스템 ⭐ (핵심 수익 구조)
- **토큰**: 유료 충전 사이버머니 (현재 관리자 수동충전, PG는 미연동)
- **포인트**: 프로모션 지급, 유효기간 있음(1/10/30일)
- **차감 순서**: 포인트 먼저 → 토큰 (FIFO로 만료 임박 포인트부터)
- 파트너 대시보드 "지갑" 탭: 잔액 + 거래내역 + 충전 안내
- 관리자 "지갑 관리" 탭:
  - 가격 설정 (지정 열람가 5만 / 공개 열람가 1만 / 월 구독료 5만 - 기본값)
  - 토큰 수동 충전 (개별)
  - **신규가입 자동 보너스 캠페인** (기간 설정 → 그 기간 승인 파트너에게 자동 지급)
  - **일괄 포인트 지급** (체크박스 다중선택 + 지역별/등급별 필터)
  - 파트너 잔액 현황

### 견적 리드 (파트너 대시보드)
- 지정 견적 / 공개 견적 하위 탭 분리
- 지정 견적은 강조 표시(테두리+뱃지) + 건수 뱃지
